Capstone Project: Milestone 1 – Build a Vulnerable Web Application
Overview
The Capstone Project serves as a culminating experience for the course, giving you the opportunity to apply everything you've learned about web application security in a meaningful and practical way. Over four weeks, your team will build, attack, defend, and present a complete web application security project.

Milestone 1 is the foundation phase. Your team will design and build a full-stack web application with intentional security vulnerabilities. This application will be penetration tested by another team in Milestone 2, so you will need to think strategically about both how vulnerabilities are introduced and how easily they can be discovered and exploited. The goal is to simulate how real-world security flaws emerge in production systems through oversight, incomplete controls, and insecure coding patterns.

Full technical details, a boilerplate starter template, an AI Prompt Library, and a Vulnerability Implementation Guide are available in Canvas Files.

Instructions

1. Team Planning

Your team has been pre-assigned. 
Meet as a team to brainstorm your application concept, define 3–5 core features, and select 6+ vulnerabilities to implement
Assign roles: Lead Developer, Database Designer, Frontend Designer, Security Tester
Document your team's plan.
2. Technical Setup

Build a full-stack application using Next.js 14+ (frontend), Flask 3.0+ (backend), SQLite 3 (database)
All components must run locally on localhost
Verify your environment is working before your first build session
Setup instructions and a boilerplate template are in Canvas Files
3. Application Design

Build any original web application concept your team chooses; no two teams should build identical apps
Required features: user authentication, user profiles, user-generated content (posts, comments, etc.), search/filter, state-changing actions (create, update, delete), minimum 3 pages, at least one API endpoint per feature
Use AI tools (ChatGPT, Claude) to accelerate development. Sample prompts are in the AI Prompt Library on Canvas
4. Vulnerability Implementation

Your application must include at least 6 exploitable vulnerabilities
Required (must include all three): SQL Injection, Cross-Site Scripting (XSS), and one Authorization Flaw (IDOR or broken access control)
Choose 3+ additional from: CSRF, File Upload flaw, Session Management flaw, Information Disclosure, Broken Authentication, Command Injection, or Sensitive Data Exposure
Each vulnerability must be documented in your PROJECT_SUMMARY.pdf with its location in code, how it works, and an example exploit payload
5. Testing & Verification

Confirm all features work and the app runs without crashes on a fresh install
Verify each vulnerability is actually exploitable using basic payloads
Have at least one team member test the full setup on a clean machine before submitting
Deliverables

Submit a single ZIP file named TeamName_Milestone1.zip containing:

Source code — full frontend and backend with README.md
README.md — setup instructions, prerequisites, how to run the app, and default credentials for testing
PROJECT_SUMMARY.pdf (3–5 pages) — project overview, features implemented with screenshots, vulnerability documentation for each of the 6+, and development notes
One team member submits on behalf of the group. List all team member names in the submission comments.

Due: Sunday, March 30, 2026 @ 11:59 PM Late submissions: –10% per day (standard course policy)
